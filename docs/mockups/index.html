🪶 birds.gaylon.photos — UI Mockups (V2)

Static HTML mockups for the V2 design: gallery is link-out only (photos stay on gaylon.photos — real CDN thumbnails are hotlinked below), Postgres + madonnahist infrastructure patterns, and first-class responsive support for iPhone / iPad / desktop. The V1 design doc (birds-app-design.md) is deprecated; see birds-app-design-V2-Fable-revision-plan.md for the V2 spec direction.

Screens

Dashboard / Near Me
Home radius, live Google Map (same key/Map ID as gaylon.photos), needs reported nearby with distance + counts + times, life-list stats.
Targets / Needs Explorer
Hancock County, ME trip scenario: "Rare this week" notable feed, your needs with recent activity, family filters, ranked best-places list with add-to-trip. Life list synced automatically via eBird login.
Trips
Trip list + detail: route map with numbered stops, per-stop notes and needs counts, delete-trip modal (family rule: modal, never toast).
My Photos
Real gaylon.photos thumbnails grouped by species, link-out to photo pages, and the unmatched-species override workflow (with real mismatches: “Blue Winged Teal”, “Boat-tailed grackle”).
Species page (Roseate Spoonbill)
Your photo strip, recent reports near home, add-to-trip, external resources (eBird / All About Birds / Macaulay / xeno-canto).

How to review responsiveness

Mobile-first with two breakpoints. Resize the window or use Chrome DevTools device toolbar:

iPhone (< 640px)
Fixed bottom nav with safe-area inset, stacked map above list (map capped at 50vh), 48px tap targets, 18px base font.
iPad (≥ 640px)
Top nav appears, bottom nav hides, 3-column photo grid.
Desktop / iPad landscape (≥ 1024px)
Map docks to a sticky right column beside the list, 4-column photo grid.

Family compliance notes

Hand-written scoped CSS only (shared mockup.css stands in for component styles) — no Tailwind. White cards on #f8f9fa, 1px #e9ecef borders, 8px radius (gaylonphotos tokens).
WCAG AAA: all text ≥ 7:1 contrast incl. muted (#495057); every status is color + text label (NEED / SEEN / NOTABLE badges).
Destructive actions use modal confirmation (see Delete trip on the Trips screen) — never toasts.
“Data from eBird.org” attribution on every screen using eBird data. Maps are Google Maps reusing the gaylon.photos key + Map ID (needs network to render).
