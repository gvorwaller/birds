### Key Technical Insight
- **MapPicker already existed** with click→reverse-geocode→draggable-pin; the
  map-pick task was integration, not new UI. Mounting it required placing it
  *outside* the planner's GET `<form>` (nested forms are invalid HTML) and
  bridging the picked point into hidden form fields.
- **Option "b" preserves precision:** carrying exact `lat`/`lng` in the URL and
  resolving the anchor from them (label as display-only) avoids the
  name→coords→name double round-trip that loses the tapped point. A remote tap
  yields a Google **Plus Code** label (no nearby address) — cosmetic only.
- **Help link placement:** the nav `items` array feeds top-nav, drawer, AND
  bottom-nav, so adding Help there would have put it on primary tabs. Adding a
  standalone `<a href="/help">` after the drawer loop keeps it drawer-only and
  role-agnostic in one line.

  an in-repo argon2 one-off (`/tmp` can't resolve the repo's `node_modules`;
  the interactive `set-password.ts` buffers oddly here).
