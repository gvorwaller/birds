## Phase 3: version fix + photo GPS map

### version fix
`/api/health` showed `version:"dev"` because `process.env.GIT_SHA` is build-time only while the endpoint read it at runtime. Now baked into the bundle via Vite `define` (`__GIT_SHA__`, declared in app.d.ts; deploy already runs `GIT_SHA=... npm run build`). Verified: a test build with `GIT_SHA=testsha123` embeds the value in the health chunk.

### photo GPS map
- `ObsMap.svelte` extended: optional `img` (thumbnail) + `linkText` per point, new `photo` pin kind (purple). InfoWindow shows the thumbnail image + title + link; img/href passed through `encodeURI`, text via escapeHtml.

- Photos page: always-shown map card "🗺️ Where you've shot — N geotagged photos" above the species groups.

### Notes / gotchas
- First tried a collapsed `<details>` for the map; a Google map created in a `display:none` container doesn't render or bind marker clicks correctly. Switched to always-shown (consistent with Dashboard/Targets), which fixed it.
- AdvancedMarkerElement info windows use `addListener('click', ...)` (emits a deprecation warning recommending `gmp-click`, but `click` works reliably; `gmp-click` via addListener did not). Marker anchor is the bottom tip, so the clickable teardrop body is ~20px above the point.

### Verified (local)
Photo map renders FL with purple pins at the real shoot locations (Jacksonville coast + central FL); tiles + pins correct. Info-window click mechanism confirmed working on the Targets map (same code). check 0/0.
