# 2026-06-30

## Curated trip add-stop hotspot suggestions

### Summary
Fixed `td-0d56cf`: Birds trip detail now offers a curated Add a stop list of
ranked nearby eBird hotspots, instead of requiring a manual place search for
every additional stop.

### What Changed
- Added a reusable ranked-place helper that uses the same current-needs
  semantics as Home and Targets.
- Trip detail now computes suggested hotspots near the trip's located stops, or
  home when the trip has no located stops.
- Suggested stops are limited to verified eBird hotspots with locIds and exclude
  hotspots already on the trip.
- Each suggestion shows target species, need count, distance/date, Map,
  Directions, eBird hotspot badge, and a direct `+ Add` action.
- Adding a suggested hotspot saves it as a hotspot stop and persists a
  planner-target note explaining why it was suggested.

### Verification
- `npm run check` - pass
- `npm test` - pass, 7 files / 21 tests
- `npm run build` - pass
- `git diff --check` - pass
- local browser QA on `/trips/3` confirmed suggestions rendered, `+ Add` added a
  hotspot, the added hotspot disappeared from suggestions after reload, and the
  planner note was saved on the new stop.

## 24-hour eBird lookback window

### Summary
Fixed `td-eb7bac`: Birds now supports a `Last 24 hours` lookback window in the
same places that already supported 7/14/30-day eBird windows.

### What Changed
- Added shared `BACK_OPTIONS` / parsing / label helpers for eBird lookback
  windows.
- Added `Last 24 hours` to Near Me, Targets, species drilldown links, and the
  trip planner.
- Updated one-day copy to display `last 24 hours` instead of `last 1 days`,
  including trip planner summaries and no-results warnings.
- Updated help text for the trip planner window options.
- Added unit coverage for supported/unsupported window values and one-day
  labels.

### Verification
- `npm run check` - pass
- `npm test` - pass, 8 files / 24 tests
- `npm run build` - pass
- `git diff --check` - pass
- local browser QA on `/?back=1`, `/targets?back=1`, and
  `/trips/plan?place=Blue%20Hill%2C%20ME&back=1` confirmed the 24-hour option
  is present, selected, and rendered as `last 24 hours`.
