# 2026-06-14

## HTTPS enforcement: moved to zone-wide, reverted birds-only workaround

Checked all four apps over http: gaylon.photos served 200 over plain http;
gifts + madonnahist redirected http->http; only birds forced https (via the
prior day's nginx fix). None sent HSTS. Conclusion: no app forced HTTPS; they
"worked" only because users always arrive via https links.

Fix: enabled Cloudflare zone-wide **Always Use HTTPS** (SSL/TLS -> Edge
Certificates). Verified all four now 301 http->https at the edge. Covers future
apps automatically.

Then reverted the birds-only nginx CF-Visitor redirect + HSTS headers (added
2026-06-13) so birds matches the siblings and the Cloudflare zone setting is the
single source of truth for http->https. Kept the service-worker no-store block
(unrelated -- that's the SW caching fix). Verified post-revert: birds still
301s http->https (now via Cloudflare), https login passes CSRF, no HSTS header.

Decision: did NOT enable zone-wide HSTS. Marginal benefit (protects only the
first request on a hostile network) vs. sticky/irreversible downside isn't worth
it for a personal family app. Revisit if needs change.
