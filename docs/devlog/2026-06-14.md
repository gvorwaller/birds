2026-06-13) so birds matches the siblings and the Cloudflare zone setting is the

first request on a hostile network) vs. sticky/irreversible downside isn't worth
it for a personal family app. Revisit if needs change.

### Key Technical Insight
Cloudflare **Flexible TLS** means the origin always sees `http` (`$scheme=http`);
the real browser scheme arrives in the `CF-Visitor` header. If a device ever
lands on `http://`, two things break: the browser shows "not secure", and
SvelteKit's CSRF guard rejects the login POST ("cross-site POST form submissions
are forbidden") because the http Origin != the app's https origin. The correct
fix is to force the browser onto HTTPS (Cloudflare "Always Use HTTPS"), not to
touch the app. Also: a Cloudflare-edge 5xx leaves NO trace in origin nginx logs,
so "500 once, fine on refresh" with clean origin logs == transient edge blip.
