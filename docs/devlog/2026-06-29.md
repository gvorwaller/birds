as a P2 task.

### What Changed
- Near Me now accepts `?back=7|14|30`, shows a Window dropdown, and uses the

  birds reported. Expanded per-place rows also show report and bird totals.

  - `users.home_google_place_id`

  - `MapPicker` preserves `place_id`
  - home location, custom trip stops, and generated historical stops persist
    Google place IDs when Google returned one
  - Google Maps links prefer `query_place_id` / `destination_place_id`, then
    fall back to exact coordinates only
- Settings now reports last synced/fetched dates and counts for taxonomy, life
  list, gallery links, eBird cache, trips, and stops.

## Near Me count fix + persistent eBird Google place IDs

### Summary
Fixed two follow-up issues from production testing:
- `td-7051cb`: Near Me/Targets summary rows now count distinct reported
  locations and sum all birds across those locations, instead of trusting
  eBird's one-row-per-species recent nearby feed.
- `td-86b297`: eBird-originated map links can now open real Google place
  panels when a confident Google place ID is known.

### What Changed
- Added `locationCount` to the shared species activity model and changed Near
  Me/Targets rows to render `locations · birds`.
- Geo-based need summaries now enrich each species from the per-species eBird
  endpoint, matching the detail page's full location set.
- Added `backend/db/migrations/0008_ebird_location_place_ids.sql` with a
  durable `ebird_locations` table keyed by eBird `loc_id`.
- Added `$server/location-placeids` to persist eBird locations, load known
  Google IDs, and perform small capped coordinate-biased Google lookups with
  confidence scoring and a 30-day retry cooldown.
- Threaded `googlePlaceId` through Near Me, Targets, species pages, Best
  Places, trip hotspot search, and trip planner candidates/saved stops.
- Added `scripts/backfill-ebird-location-place-ids.mjs`, a dry-run-first
  audit/apply tool that seeds eBird locations from `ebird_cache` and backfills
  confident Google place IDs.

### Verification

- `npm run build` — pass
- targeted Prettier check on touched code/Svelte/script files — pass
- test DB migration applied `0008_ebird_location_place_ids.sql` cleanly
- local browser smoke on `http://127.0.0.1:5178/?back=14`:
  - summary rows showed examples like `18 locations · 45 birds`
  - Google Maps links included `query_place_id` and `destination_place_id`
  - `/targets` loaded without internal errors

  coordinates, not unsafe name-only searches.
