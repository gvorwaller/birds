### Key Technical Insight
- **eBird API exposes no free-text notes.** Live calls confirmed `ref/hotspot/info` returns
  admin metadata only (no species count, no notes), and there's no species-behavior
  endpoint — so the LLM idea was reframed: species traits come from the model's own
  (hedged) knowledge, hotspot "stats" come from the geo endpoint, weather from NWS.
- **Local dev = the PG17 test cluster (15436), matching prod (17.x).** pg17 had been
  uninstalled (a "ghost" server on a deleted binary); reinstalled `postgresql@17`, fixed
  `find_pg_bin` to also locate an unlinked Cellar install, added `npm run test:db:up`.
  Cluster isolation verified — no interference with BTC/madonnahist.
- **Multi-user seam is one nullable column.** `users.views_user_id`;
  `scopeOwnerId = views_user_id ?? user.id` → `locals.scopeId`. Reads use `scopeId`, writes
  use `user.id`. The data layer already took `userId` everywhere, so only `locals.ownerId`
  in 7 routes had to change. `#2` family-sharing = expand `readableUserIds()` + a `shares`
  table. Gallery decoupled by gating on `users.gallery_url` (no per-user photo storage).
  IDOR verified (Marcus → another user's trip → 404).
- `--env-file` populates `process.env` at runtime, so secrets don't appear in
  `/proc/<pid>/environ` — that's not a valid way to check whether a prod key loaded.
- No prettier config in repo (tabs + single quotes is the de-facto style) — run prettier
  with `--use-tabs --single-quote` or it churns whole files to spaces/double-quotes.
